# ⚖️ LPI 9.279/96 - Destaques para Inovação e Software

Estes são os critérios legais utilizados pelo Agente Themis para avaliação.

## 🚫 Art. 10 - Não é Invenção
- **V - Programas de computador em si.** (O código não é patenteável, a função técnica sim).
- **III - Métodos comerciais/negócio.** (Regras de business não são patenteáveis).

## ✅ Requisitos de Patenteabilidade
- **Art. 11 (Novidade):** Não compreendido no estado da técnica.
- **Art. 13 (Atividade Inventiva):** Não decorre de maneira óbvia para um técnico no assunto.
- **Art. 15 (Aplicação Industrial):** Pode ser produzido industrialmente.

## 💡 Diretrizes para Software (CII)
Para ser patenteável, o software deve resolver um **Problema Técnico** com uma **Solução Técnica**.
Exemplos:
- Melhoria de performance de hardware.
- Segurança de dados.
- Processamento de imagem/sinal.
